Linux相关

包括Linux内核和GNU系统工具。

Linux内核功能

  • 系统内存管理

交换空间(swap；虚拟内存)

  • 软件程序管理

系统开机初始化：/etc/inittab或/etc/init.d目录

系统开机运行级别：

  1. 关机

  2. 单root用户，无网络，无守护进程

  3. 多用户，无网络，无守护进程

  4. 正常启动

  5. 用户自定义

  6. 多用户，图形页面

  7. 重启

  • 硬件设备管理

设备文件：字符型设备（调节器和终端）、块设备（硬盘）、网络设备（网卡）

  • 文件系统管理

文件系统格式：ext4（第四代扩展文件系统，支持日志扩展）、XFS（高性能64位）

VFS(virtual file system)：虚拟文件系统，当文件系统被挂载和使用时，信息缓存在内存中

GNU（GNU’s Not Unix）

GNU coreutils：核心系统工具

  1. 处理文件工具
  2. 操作文本工具
  3. 管理进程工具

基本Shell命令

shell用户

/etc/passwd下记录了用户的基本配置信息

shell文件

常见目录：

/bin 二进制目录，存放用户GNU工具

/sbin 二进制目录，存放管理员GNU工具

/sys 系统硬件信息目录

/ boot启动目录

/lib 库目录，存放系统和应用程序主文件

/proc 进程目录

文件和目录处理

  • 文本匹配过滤：

?：一个字符

*：0个或者多个字符

[]：字母范围，如[a-j]，匹配一个在[a-j]中间的数

[!]：排除内容

  • 文件处理

touch：创建新文件，改变文件修改时间（不会对文件进行其他操作）

cp：复制粘贴

  • 文件链接

符号链接：ln -s 创建符号链接（相当于快捷方式）

硬链接：同一个文件的不同访问形式

只能在统一存储媒体创建硬链接，否则只能符号链接

  • 文件移动

mv只会影响文件名，时间戳和inode没有改变

  • 查看文件

file：查看文件类型

cat ：显示文件

more：回车下一行、空格下一页、control+b上一页、b回到第一

less：同more

tail：显示末尾几行

  • cp -i -R （i若覆盖文件，则强制询问、R递归复制）

  • cat -n(-b) （n显示每行行号，b显示有文字行号）

  • file

  • tail -n 15 （显示到15行）

进阶shell

进程相关

进程分为实时进程和非实时进程；

非实时进程划分为：交互式进程、批处理进程、实时进程。

进程调度CFS算法。

  • 查看所有进程（常见）

显示当前进程：ps -f

标准格式显示：ps -ef

BSD格式显示：ps -aux

按顺序：top

PID 进程的ID
PPID 父进程号
USER 进程属主名
PR 进程优先级，PRIORITY：动态优先级，取值0-31
0-15普通优先级，高优先级进程优先运行，优先级相同按照时间片划分
16-31实时优先级，不按照时间片轮转，相同优先级不按照时间片轮转
若为RT，则表示实时策略
NI 进程谦让度值，NICE：静态优先级，取值-10~19
NI值越小，进程优先级越高
VIRT 进程占用虚拟内存
RES 进程占用物理内存
SHR 进程和其他进程共享内存量
S 进程的状态（D：可中断休眠；R：正在运行；S：休眠；Z：僵死；T：跟踪或停止）
%CPU 所占用CPU时间比例
%MEM 所占可用内存比例
COMMAND 启动的程序名

杀死进程：

kill pid

killall：killall http*

磁盘空间

挂载磁盘：mount

卸载磁盘：umount （umount /mnt）

磁盘存储情况：df -h

特定目录的使用情况：du -h （du -sh *）

处理文件

排序：sort -n(-t)(-r) file（n按照字符串数字排序；t以制表符作为空格对象；r反序输出）

搜索数据：grep -v(-e) （v不含；e多个”或”匹配）

压缩数据：gzip/gunzip（不常用了）；tar -zcvf/tar -zxvf ；zip/unzip

涉及命令

  • ps
  • top
  • mount
  • umount
  • df -h
  • du -sh *
  • du -sh * | sort -nr
  • grep -v

子shell

环境变量

存储shell对话和工作环境信息。

分为全局环境变量、局部环境变量。

设置环境变量

查看全局变量：printenv (HOME)或者env； echo $HOME

查看局部变量：set；局部变量在父shell和子shell中不共享

另外： 父shell若设置了变量，但未导出“export”，则视为局部变量

设置全局环境变量：export（将在子shell中可用，不用加$符号）

my_variable=”I am a golobal variable” ; export my_variable ; echo $my_variable

删除环境变量：unset my_variable（子shell删除后，依然在父shell中可用）

ps：$不只是变量，还会讲变量作为命令行参数；

如果需要用到变量，则加$符号，如果需要操作变量，则不加$；env显示变量值除外。

默认shell环境

变量 描述
HOME 当前用户主目录
MAIL 用户收件箱地址
PATH Shell命令查找目录列表
PWD 工作目录
LANG 语言环境类别
SHLVL shell层级

内建命令通过PATH来寻找对应的程序，中间用冒号分分隔即可

增加路径： PATH=$PATH:/home/script；对PATH的修改在退出或者重启系统后失效

登录shell

登录shell涉及以下文件：

文件 说明
/etc/profile 默认bash shell启动主文件，任何用户登录都会
$HOME/.bash_profile 优先级1
$HOME/.bash_login 优先级2
$HOME/.profile 优先级3
$HOME/.bashrc 交互式shell启动

环境变量持久化：

最好在/etc/profile下创建.sh结尾的文件，将修改后的变量存放其中

PS：若将alias设置放在$HOME/.bashrc中，实现永久化

数组变量

mytest=(1 2 3 4 5)

echo $mytest

echo ${mytest[2]}

注意：可以用unset删除数组中某个元素，但是删除后，该处索引仍存在！

unset mytest[2] >

echo ${mytest[*]} > 1 2 4 5

echo ${mytest[2]} > 此处为空！！并不是4！！！！

  • printenv
  • env
  • set
  • export
  • unset
  • test=(1 2 3 4 5)
  • echo ${test[*]}
